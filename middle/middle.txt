- ORM (Object-Relational Mapping): Django использует ORM,
что позволяет взаимодействовать с базами данных с помощью Python-кода,
а не SQL-запросов. Это делает разработку быстрее и безопаснее.
- Поддерживаемые базы данных: Django поддерживает PostgreSQL, MySQL, SQLite и Oracle.
Выбор базы данных зависит от требований проекта (масштабируемость, сложность, стоимость).
- Настройка базы данных: Настройки подключения к базе данных (имя, пользователь, пароль,
хост) указываются в файле settings.py в переменной DATABASES.
- Модели: Модели Django описывают структуру таблиц в базе данных.
Каждая модель – это класс Python, который наследуется от  django.db.models.Model.
Атрибуты модели – это поля таблицы.
- Миграции: Django использует миграции для создания и изменения структуры базы данных
на основе моделей. Миграции позволяют безопасно и контролируемо вносить изменения в
базу данных. Чтобы создать миграции используется команда python manage.py makemigrations,
а применить – python manage.py migrate.
- manage.py : Это утилита командной строки, которая упрощает управление проектом Django
(запуск сервера, создание миграций, создание суперпользователя и т.д.).
- settings.py : Важнейший файл конфигурации проекта, где определяются настройки базы данных,
 часовой пояс, языки, установленные приложения и множество других параметров.

Этапы взаимодействия:

1. Клиент (браузер) отправляет HTTP-запрос на сервер (Django).
Запрос может быть на получение страницы, отправку данных формы и т.д.
2. Сервер (Django) получает запрос, обрабатывает его. Он определяет,
какой код должен быть выполнен для данного запроса (какое представление (view) нужно вызвать).
3. Представление (view) Django взаимодействует с базой данных.
Оно может получать данные из базы данных, сохранять данные в базу данных,
изменять данные и т.д.  Для этого используется ORM Django.
4. База данных возвращает данные серверу.
5. Сервер (Django) формирует HTTP-ответ.  Ответ может содержать HTML-код страницы,
данные в формате JSON и т.д. Он может использовать шаблоны для динамической генерации HTML.
6. Сервер отправляет HTTP-ответ клиенту (браузеру).
7. Клиент (браузер) отображает полученный ответ. Если это HTML-код,
то браузер отображает страницу, если это JSON, то браузер может использовать его
для обновления части страницы.

Примеры популярных веб-приложений, созданных на Django:

1. Instagram: Хотя сейчас используется более сложная архитектура, изначально Instagram был создан на Django. Django помог им быстро разработать и запустить проект.
2. Mozilla: Использует Django для различных разделов своего сайта, включая систему управления дополнениями для Firefox.
3. Disqus: Популярный сервис комментариев, полностью построенный на Django.
